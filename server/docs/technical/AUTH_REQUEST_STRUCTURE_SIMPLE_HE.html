AUTH - מבנה בקשות בטבלאות

מטרה: להבין מהר מה כל endpoint צריך לקבל בבקשה (Request).

Base Path: /api/auth

טבלת על - כל endpoints של AUTH

# Endpoint Method Authorization Token - האם נדרש Token - מיקום Headers חובה URL Params Query Body מה עובר ולידציה Middleware שגיאות נפוצות דוגמת Success Response שכבות (בקצרה)
1 /register POST לא לא אין Content-Type: application/json אין אין יש registerSchema (body) authRateLimiter 400, 409, 429 { success, data: { user, token, refreshToken }, message } authRateLimiter -> AuthController.register -> AuthService.register
2 /login POST לא לא אין Content-Type: application/json אין אין יש loginSchema (body) authRateLimiter 400, 401, 423, 429 { success, data: { user, token, refreshToken }, message } authRateLimiter -> AuthController.login -> AuthService.login
3 /forgot-password POST לא לא אין Content-Type: application/json אין אין יש forgotPasswordSchema (body) authRateLimiter 400, 429 { success, message } authRateLimiter -> AuthController.forgotPassword -> AuthService.forgotPassword
4 /reset-password/:token POST לא לא token הוא URL param, לא JWT auth Content-Type: application/json token (חובה) אין יש resetPasswordSchema (body) + token check (params) authRateLimiter 400 { success, message } authRateLimiter -> AuthController.resetPassword -> AuthService.resetPassword
5 /refresh POST לא כן body.refreshToken Content-Type: application/json אין אין יש בדיקה ידנית ב-controller: refreshToken חובה authRateLimiter 400, 401, 429 { success, message, data: { token, refreshToken } } authRateLimiter -> AuthController.refreshToken -> AuthService.refreshAccessToken
6 /google POST לא כן body.idToken Content-Type: application/json אין אין יש בדיקה ידנית ב-controller: idToken חובה authRateLimiter 400, 403, 429 { success, data: { user, token, refreshToken }, message } authRateLimiter -> AuthController.googleLogin -> findOrCreateGoogleUser
7 /verify GET כן כן header Authorization: Bearer <access_token> Authorization: Bearer <access_token> אין אין אין authenticate middleware + verifyToken authenticate 401 { success, data: { user } } authenticate -> AuthController.verify -> AuthService.verifyToken
8 /profile GET כן כן header Authorization: Bearer <access_token> Authorization: Bearer <access_token> אין אין אין authenticate middleware authenticate 401 { success, data: { user } } authenticate -> AuthController.getProfile -> AuthService.getProfile
9 /profile PUT כן כן header Authorization: Bearer <access_token> Authorization + Content-Type אין אין יש authenticate middleware + updateProfileSchema (body) authenticate 400, 401 { success, data: { user }, message } authenticate -> AuthController.updateProfile -> AuthService.updateProfile
10 /change-password POST כן כן header Authorization: Bearer <access_token> Authorization + Content-Type אין אין יש authenticate middleware + changePasswordSchema (body) authenticate 400, 401 { success, message } authenticate -> AuthController.changePassword -> AuthService.changePassword
11 /logout POST כן כן header Authorization: Bearer <access_token> Authorization: Bearer <access_token> אין אין אין authenticate middleware authenticate 401 { success, message } authenticate -> AuthController.logout -> AuthService.logout

טבלאות Body לפי Endpoint

1) POST /register

שדה סוג חובה/רשות כלל ולידציה
name string חובה לפחות 2 תווים
email string חובה פורמט אימייל תקין
password string חובה לפחות 6 תווים

דוגמה: { "name": "Shmuel", "email": "shmuel@example.com", "password": "SecurePass123!" }

2) POST /login

שדה סוג חובה/רשות כלל ולידציה
email string חובה פורמט אימייל תקין
password string חובה לא ריק

דוגמה: { "email": "shmuel@example.com", "password": "SecurePass123!" }

3) POST /forgot-password

שדה סוג חובה/רשות כלל ולידציה
email string חובה פורמט אימייל תקין

דוגמה: { "email": "shmuel@example.com" }

4) POST /reset-password/:token

רכיב שדה סוג חובה/רשות כלל ולידציה
params token string חובה נשלח בתוך ה-URL
body password string חובה לפחות 6 תווים
body confirmPassword string חובה חייב להיות זהה ל-password

דוגמה URL: POST /api/auth/reset-password/abc123-reset-token

דוגמה Body: { "password": "NewSecurePass123!", "confirmPassword": "NewSecurePass123!" }

5) POST /refresh

שדה סוג חובה/רשות כלל
refreshToken string חובה חייב להישלח בבקשה

דוגמה: { "refreshToken": "eyJhbGciOi..." }

6) POST /google

שדה סוג חובה/רשות כלל
idToken string חובה Google ID Token

דוגמה: { "idToken": "google-id-token-from-client" }

7) GET /verify

רכיב שדה חובה/רשות ערך צפוי
header Authorization חובה Bearer <access_token>

8) GET /profile

רכיב שדה חובה/רשות ערך צפוי
header Authorization חובה Bearer <access_token>

9) PUT /profile

שדה סוג חובה/רשות כלל ולידציה
name string רשות לפחות 2 תווים
email string רשות פורמט אימייל תקין

כלל חשוב: חייב להיות לפחות שדה אחד בבקשה (name או email).

דוגמה: { "name": "Shmuel K" }

10) POST /change-password

שדה סוג חובה/רשות כלל ולידציה
currentPassword string חובה לא ריק
newPassword string חובה לפחות 6 תווים
confirmPassword string חובה חייב להיות זהה ל-newPassword

דוגמה: { "currentPassword": "OldPass123!", "newPassword": "NewPass123!", "confirmPassword": "NewPass123!" }

11) POST /logout

רכיב שדה חובה/רשות ערך צפוי
header Authorization חובה Bearer <access_token>
